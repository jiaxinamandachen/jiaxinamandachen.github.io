# Project instructions

- After every completed website update, verify the affected page or functionality.
- Commit all intended project changes to Git.
- Push the commit to the configured `origin` remote before reporting completion.
- If the remote branch has changed, fetch and rebase safely before pushing. Never force-push or overwrite remote work.
- If pushing is blocked by authentication, conflicts, or network errors, report the blocker clearly instead of claiming the update is complete.
